📄 Import Samples — JSON & CSV

Copy these examples to create your own test data files. Each task type requires specific fields. Both JSON and CSV formats are accepted; CSV files are converted to JSON on import.

Classification Scenarios

The model receives customer_input and must predict category, subcategory, priority and sentiment.

Required fields

id customer_input expected_category expected_subcategory expected_priority expected_sentiment context (optional)

JSON format

[
  {
    "id": "CLASS_001",
    "customer_input": "My bill is almost double what I usually pay. Can you explain these charges?",
    "expected_category": "billing_inquiry",
    "expected_subcategory": "unexpected_charges",
    "expected_priority": "high",
    "expected_sentiment": "frustrated",
    "context": "{\"account_tenure_months\": 18, \"billing_cycle\": \"2026-01\"}"
  },
  {
    "id": "CLASS_002",
    "customer_input": "I have had no mobile signal since last night's storm.",
    "expected_category": "technical_support",
    "expected_subcategory": "outage_or_service_disruption",
    "expected_priority": "critical",
    "expected_sentiment": "very_angry",
    "context": "{\"network_type\": \"mobile_4g\", \"outage_ticket_open\": true}"
  }
]

CSV format

The context field is a JSON string — wrap it in double-quotes and escape inner quotes.

id,customer_input,expected_category,expected_subcategory,expected_priority,expected_sentiment,context
CLASS_001,"My bill is almost double what I usually pay. Can you explain these charges?",billing_inquiry,unexpected_charges,high,frustrated,"{""account_tenure_months"": 18, ""billing_cycle"": ""2026-01""}"
CLASS_002,"I have had no mobile signal since last night's storm.",technical_support,outage_or_service_disruption,critical,very_angry,"{""network_type"": ""mobile_4g"", ""outage_ticket_open"": true}"

Dialog Scenarios

Tests the model's ability to identify context gaps and generate an optimal follow-up in a multi-turn conversation.

Required fields

id conversation context_gaps optimal_follow_up follow_up_rules expected_resolution_turns

JSON format

[
  {
    "id": "DLG_001",
    "conversation": "[{\"role\": \"customer\", \"message\": \"Hi, my mobile bill is way higher than usual this month.\"}]",
    "context_gaps": "customer authentication details | account or phone number | billing period",
    "optimal_follow_up": "I'm sorry about the confusion. Could you provide your account number and the billing month?",
    "follow_up_rules": "Begin with empathy | Request minimum necessary details | Avoid assuming the cause",
    "expected_resolution_turns": 2
  },
  {
    "id": "DLG_002",
    "conversation": "[{\"role\": \"customer\", \"message\": \"Every evening my internet gets super slow.\"}, {\"role\": \"agent\", \"message\": \"I'm sorry to hear about the slowdown.\"}]",
    "context_gaps": "connection type | modem/router model | Wi-Fi vs wired | speed test results",
    "optimal_follow_up": "Could you tell me your connection type and run a speed test during the slowdown?",
    "follow_up_rules": "Clarify technical environment first | Ask for concrete measurements | Keep instructions simple",
    "expected_resolution_turns": 3
  }
]

Tool Calling Scenarios

Tests function/tool calling: the model must pick the right tool(s) and arguments for a given query.

Required fields

id query available_tools expected_tool_calls expected_parameters

JSON format

[
  {
    "id": "TC_001",
    "query": "What's the weather like in Madrid right now?",
    "available_tools": "[{\"type\": \"function\", \"function\": {\"name\": \"get_current_weather\", \"description\": \"Get current weather for a location\", \"parameters\": {\"type\": \"object\", \"properties\": {\"location\": {\"type\": \"string\"}, \"unit\": {\"type\": \"string\", \"enum\": [\"celsius\", \"fahrenheit\"]}}, \"required\": [\"location\"]}}}]",
    "expected_tool_calls": "get_current_weather",
    "expected_parameters": "{\"location\": \"Madrid, Spain\", \"unit\": \"celsius\"}"
  },
  {
    "id": "TC_002",
    "query": "Book a flight from NYC to London and a hotel for 3 nights.",
    "available_tools": "[{\"type\": \"function\", \"function\": {\"name\": \"search_flights\", \"description\": \"Search flights\", \"parameters\": {\"type\": \"object\", \"properties\": {\"origin\": {\"type\": \"string\"}, \"destination\": {\"type\": \"string\"}}, \"required\": [\"origin\", \"destination\"]}}}, {\"type\": \"function\", \"function\": {\"name\": \"search_hotels\", \"description\": \"Search hotels\", \"parameters\": {\"type\": \"object\", \"properties\": {\"city\": {\"type\": \"string\"}, \"nights\": {\"type\": \"integer\"}}, \"required\": [\"city\"]}}}]",
    "expected_tool_calls": "search_flights | search_hotels",
    "expected_parameters": "{\"search_flights\": {\"origin\": \"New York\", \"destination\": \"London\"}, \"search_hotels\": {\"city\": \"London\", \"nights\": 3}}"
  }
]

💡 Tips

  • You don't need all 5 task types — import at least one prompt + one matching data file.
  • CSV files are automatically converted to JSON on import. The canonical storage format is JSON.
  • Fields containing commas, quotes or newlines must be wrapped in double-quotes. Escape inner quotes by doubling them: "" .
  • The id field should be unique within each file (e.g. CLASS_001, DLG_001, RAG_001…).
  • JSON fields embedded in CSV (like context, conversation, available_tools) are stored as strings — they'll be parsed at evaluation time.

🧠 Prompt Generation Best Practices

When you use ✨ AI Generate on the Prompts page, the system automatically applies model-specific best practices to produce optimised prompts. Here's how it works.

📐 Two-Tier Guidance Architecture

The prompt generator uses a centralised guidance module (model_guidance.py) with two tiers that are merged at generation time:

Tier 1 — Family Guidance

Base best practices shared by all models in a family.

GPT-4.x — Explicit CoT, Markdown tables, verbose instructions, JSON examples, max_tokens, system role
GPT-5.x — Native reasoning (no CoT), YAML schemas, concise/declarative, max_completion_tokens, developer role
Mistral — Explicit CoT, few-shot examples, multilingual, system role
Gemini — Explicit CoT, reasoning_effort, no seed support, multimodal
Phi (SLM) — Very explicit instructions, CoT needed, 16K context limit, <|im_start|> tokens
Realtime — Voice/session instructions, bullet format, VAD, short spoken turns

Tier 2 — Deployment-Specific Addendum

Per-model nuances layered on top of the family guidance.

GPT-4.1 — Strongest instruction follower, 1M context, agentic, #inner_thoughts pattern
GPT-4o — Speed-optimised, multimodal, strong few-shot learner
GPT-5.1 — Reasoning model, declarative prompts, no temperature, reasoning_effort
GPT-5.4-mini — Cost-effective GPT-5, native shallow reasoning, concise prompts
Mistral-Large-3 — 128K context, multilingual, parallel tool calling
Gemini 3 Flash — 1M context, multimodal, reasoning_effort, low-latency
Phi-4 — 14B SLM, math/GPQA strong, 16K context, MIT licensed

The function get_guidance() merges both tiers using: exact match on deployment name → longest prefix match → family-only fallback.

⚙️ How Prompt Generation Works

When you click ✨ AI Generate, the PromptManager builds a meta-prompt that is sent to the generator LLM. This meta-prompt includes:

  1. Two-tier model guidance — the combined family + deployment-specific best practices from model_guidance.py
  2. Model identity — the model label and its concrete deployment name (e.g. "GPT-5 (deployment: gpt-5.4)")
  3. Task description — what the task type requires (classification, dialog, RAG, tool calling)
  4. Reference prompt snippet — an excerpt from the existing prompt as starting context
  5. Canonical JSON schema — the 7 mandatory output fields all models must produce for uniform evaluation
  6. Taxonomy constraints — shared categories and subcategories from the canonical model
  7. Domain tool summaries — available tools and their schemas (for tool-calling tasks)

The result: each model gets a materially different prompt optimised for its specific capabilities. For example, GPT-4o prompts are ~339 lines with explicit CoT, while GPT-5 prompts are ~236 lines with declarative instructions leveraging native reasoning.

🔑 Key Principles Behind the Guidance

  • Models with native reasoning (GPT-5.x, o-series) should receive declarative, goal-focused prompts — explicit Chain-of-Thought hurts more than helps.
  • Models without native reasoning (GPT-4.x, Mistral, Gemini, Phi) benefit from explicit step-by-step CoT instructions and concrete few-shot examples.
  • Small Language Models (Phi-4, 14B) need the most explicit guidance — every token counts in a 16K context window.
  • API parameter differences are critical: max_tokens vs max_completion_tokens, system vs developer role, seed support — mismatches cause silent failures.
  • Reproducibility via seed: the seed parameter enables deterministic outputs for fair evaluation comparisons — GPT-4.x, GPT-5.x and Mistral support it, but Gemini and Phi-4 do not, so for those models we rely on temperature=0.1 as a best-effort alternative to minimise response variability.
  • The guidance is maintained in a single source of truth (model_guidance.py) consumed by both the AI prompt generator and the topic import tool.
